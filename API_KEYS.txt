================================================================================
  RABBIT HOLE - ONDE PEGAR E ONDE COLOCAR API KEYS
================================================================================

IMPORTANTE: Nunca commite arquivos com chaves reais. Use .env.local (já está
no .gitignore). Este arquivo é só um guia.

--------------------------------------------------------------------------------
ONDE COLOCAR AS CHAVES
--------------------------------------------------------------------------------

1. Crie um arquivo na RAIZ do projeto chamado:

   .env.local

   Caminho completo (exemplo Windows):
   E:\Projetos\RabbitHole\RabbitHole\.env.local

   Caminho completo (exemplo Mac/Linux):
   /seu-usuario/Projetos/RabbitHole/RabbitHole/.env.local

2. Dentro do .env.local, coloque UMA variável por linha, no formato:

   NOME_DA_VARIAVEL=valor_da_chave

   Exemplo:
   UNSPLASH_ACCESS_KEY=sua_chave_aqui

3. Reinicie o servidor de desenvolvimento (npm run dev) após alterar .env.local.

--------------------------------------------------------------------------------
APIs USADAS NO PROJETO
--------------------------------------------------------------------------------

--- WIKIDATA (busca de entidades e relações) ---
  Precisa de API key? NAO (uso público).
  Onde configurar User-Agent (opcional, recomendado em produção):
  - No código: src/lib/wikidata.ts (header User-Agent nas requisições).
  - Você pode definir em .env.local: WIKIDATA_USER_AGENT=RabbitHole/1.0 (seu@email.com)
  Links:
  - API: https://www.wikidata.org/w/api.php
  - Documentação: https://www.wikidata.org/wiki/Wikidata:Data_access
  - Política de User-Agent: https://www.mediawiki.org/wiki/Wikidata_Query_Service/User_Agent_Policy

--- WIKIPEDIA (resumos e imagens) ---
  Precisa de API key? NAO (uso público).
  Onde configurar User-Agent (opcional):
  - No código: src/lib/wikipedia.ts (header User-Agent).
  Links:
  - API REST: https://en.wikipedia.org/api/rest_v1
  - Documentação: https://www.mediawiki.org/wiki/API:Main_page

--- UNSPLASH (imagens - opcional) ---
  Precisa de API key? SIM (para uso além do placeholder atual).
  Onde colocar a chave: .env.local
  Nome da variável sugerida: UNSPLASH_ACCESS_KEY

  Onde pegar a chave:
  1. Acesse: https://unsplash.com/developers
  2. Clique em "Register as a developer" ou "Your apps".
  3. Crie uma aplicação (New Application).
  4. Aceite os termos.
  5. Na página do app, copie "Access Key" (e opcionalmente "Secret" se for usar em backend).
  Link direto (login): https://unsplash.com/oauth/applications

  Tutorial rápido:
  - Documentação: https://unsplash.com/documentation
  - Exemplo de uso: GET https://api.unsplash.com/search/photos?query=termo&client_id=SUA_ACCESS_KEY

  No projeto: atualmente as imagens usam placeholders. Para usar a API de verdade,
  será preciso alterar src/lib/unsplash.ts para usar process.env.UNSPLASH_ACCESS_KEY
  nas chamadas à API do Unsplash.

--- GEMINI (Google) - opcional, para enriquecer conteúdo ---
  Precisa de API key? SIM (opcional).
  Onde colocar: .env.local
  Nome da variável: GEMINI_API_KEY

  Modelo recomendado no projeto: gemini-1.5-flash (bom custo-benefício e baixa latência).

  Onde pegar: https://aistudio.google.com/app/apikey
  Documentação: https://ai.google.dev/docs

  Uso no Rabbit Hole:
  - Geração de "fato interessante" (interestingFact) ao enriquecer entidades no discovery.
  - Se GEMINI_API_KEY estiver definida, o texto é enviado ao Gemini após a Wikipédia;
    o resultado só substitui interestingFact quando a Wikipédia não tiver um trecho bom.
  - Implementação: src/lib/gemini.ts (generateInterestingFact) e src/lib/discovery.ts (enrichEntityWithWikipedia).

  Se você expôs a chave em algum lugar (ex.: chat), revogue em https://aistudio.google.com/app/apikey e crie uma nova.

CHECKLIST DE SEGURANÇA
--------------------------------------------------------------------------------

[ ] Nunca commitar .env.local ou .env (já estão no .gitignore).
[ ] Não colocar chaves em arquivos .txt com nomes como "minhas_chaves.txt" no repo.
